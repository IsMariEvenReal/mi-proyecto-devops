# mi-proyecto-devops
Práctica para Fundamentos de DevOps. Control de Versiones y Flujo DevOps con GitHub.

DESCRIPCIÓN DEL PROYECTO:
Página web estática como práctica para control de versiones, que utiliza GitHub como plataforma DevOps. Es una página web que utiliza solamente textos de ejemplo.

OBJETIVO:
Reforzar mi aprendizaje actual en Fundamentos de DevOps, que comprende temas del uso de Git y GitHub como control de versiones.

ESTRUCTURA DEL REPOSITORIO:

mi-proyecto-devops/
│── src/
│   ├── index.html
│   ├── styles.css
│── scripts/
│   └── deploy.sh
│── README.md
│── .gitignore

FLUJO DE TRABAJO UTILIZADO:

El flujo de trabajo siguió el ciclo a continuación:

- Codificación
- Pruebas
- Gestión de versiones (esto incluye commits, push, pull requests, entre otros)
- Despliegue

COMANDOS GIT PRINCIPALES:

- Add
- Commit
- Push
